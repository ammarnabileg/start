## نظرة عامة على التطبيق

تطبيق التاجر = **"أدِر برنامج الولاء الخاص بمتجرك"**. الكاشير يمسح كود العميل، يسجّل زيارة، يضيف نقاط، يستبدل مكافآت. الأدوار: `merchant_owner` (المدير) و `staff` (كاشير/مدير فرع).

**أهم شاشة:** **الماسح + ملف العميل** — لازم سريعة وبأقل ضغطات. الكاشير عنده طابور، فكل ضغطة زيادة = احتكاك. خلّي "إضافة نقاط" و"تسجيل زيارة" على بُعد ضغطة-اتنين بعد المسح.

### التنقّل (Bottom Tab Bar — 4 تابات)
| # | التاب | الأيقونة | الوظيفة |
|---|---|---|---|
| 1 | **لوحة التحكم** | داشبورد | نظرة سريعة + أرقام |
| 2 | **المسح (Scanner)** | كاميرا (زر كبير في النص) | مسح كود العميل |
| 3 | **الإدارة** | إعدادات | مكافآت/زيارات/مستويات/كوبونات/فروع/موظفين |
| 4 | **حسابي** | متجر | بيانات النشاط + الاشتراك |

### الصلاحيات (Role-Based)
| الدور | يقدر يعمل |
|---|---|
| **الكاشير (cashier)** | مسح + تسجيل زيارة + إضافة نقاط + استبدال + تطبيق كوبون فقط |
| **مدير الفرع (branch_manager)** | كل ما سبق + إدارة فرعه + موظفين فرعه |
| **المدير (manager/owner)** | كل حاجة في المتجر (إعداد + تحليلات + موظفين + اشتراك) |

> الصلاحيات تُفرض على السيرفر (RLS + Edge Functions)، مش مجرد إخفاء أزرار في الـ UI.

## الشاشات (Screen-by-Screen)

### 2.1 — Splash
- token صالح → لوحة التحكم. مفيش → الترحيب. (نفس منطق العميل.)

### 2.2 — شاشة الترحيب
- لوجو + "أدِر برنامج الولاء الخاص بمتجرك" + زرارين: "تسجيل نشاط جديد" / "تسجيل الدخول".

### 2.3 — تسجيل النشاط (Register Business)
| الحقل | Label |
|---|---|
| اسم النشاط | اسم المتجر / النشاط |
| نوع النشاط | قائمة (مطعم / كافيه / متجر / صالون / أخرى) |
| رقم الجوال + البريد | — |
| السجل التجاري | رقم السجل التجاري (اختياري) |
| الشعار | شعار المتجر (رفع صورة — تُضغط قبل الرفع) |
| العنوان | عنوان المتجر (نص + اختيار من الخريطة لتحديد lat/lng) |
| كلمة المرور | — |
| الشروط | checkbox |

- زرار "إرسال الطلب" → **OTP** → **شاشة "قيد المراجعة"**.
- **تقني:** ينشئ صف في `merchants` بـ `status = pending`.

### 2.4 — قيد المراجعة (Pending Approval)
- **ناس كتير بتنسى الشاشة دي وبتسيب التاجر تايه.**
- أيقونة ساعة + "طلبك قيد المراجعة" + "سنخطرك فور الموافقة على حسابك. عادةً خلال 24-48 ساعة." + زرار "تحديث الحالة".
- **UX:** أول ما الأدمن يوافق → إشعار + الشاشة تتحوّل لاختيار الباقة.
- **تقني:** Realtime subscription على صف الـ merchant، أو polling عند "تحديث الحالة".

### 2.5 — تسجيل الدخول
- رقم الجوال/الإيميل + كلمة المرور + "نسيت كلمة المرور؟".

### 2.6 — اختيار الباقة (Plans / Subscription)
- كروت الخطط:
  - **تجربة مجانية:** "مجانًا لمدة 30 يومًا — بكل المزايا." (مميّزة)
  - **شهري:** $9/شهر + المزايا.
  - **سنوي:** $99/سنة + شارة "وفّر ~8%".
- أزرار: "ابدأ التجربة المجانية" / "اشترك".
- **مهم (النسخة الأولى):** الدفع **يدوي** — الزرار يبدأ التجربة، والاشتراك المدفوع **يفعّله الأدمن** بعد التحصيل (تحويل/فاتورة). الشاشة جاهزة لبوابة الدفع بعدين بدون إعادة تصميم.

### 2.7 — لوحة التحكم (Dashboard) ⭐ الرئيسية
- **فلتر الفرع فوق** (لو فيه فروع): "كل الفروع ▾".
- **بطاقات الأرقام:** عدد العملاء · زيارات اليوم/الأسبوع · إجمالي النقاط الموزّعة · المكافآت المُستبدلة · معدّل العودة (%).
- **زرار كبير "مسح رمز العميل"** (أو هو تاب الـ Scanner).
- **آخر النشاطات:** قائمة سريعة (فلان ربح نقاط / فلان استبدل مكافأة) بالوقت.
- **تنبيه الاشتراك:** لو التجربة قربت تخلص → بانر "تنتهي تجربتك خلال 5 أيام — اشترك للمتابعة."

### 2.8 — الماسح (QR Scanner) ⭐ أكتر شاشة استخدامًا
- نافذة الكاميرا + إطار مسح في النص + "وجّه الكاميرا نحو رمز العميل" + زرار فلاش.
- **الحالات:**
  - نجاح → ينقل فورًا لـ "ملف العميل" (2.9).
  - كود غير صالح/منتهي → "الرمز غير صالح، اطلب من العميل تحديث الشاشة." (بسبب الـ QR المتغيّر).
  - مفيش إذن كاميرا → شاشة تمهيد + "السماح بالكاميرا".
- **تقني:** `mobile_scanner` يقرأ payload → يستدعي Edge Function `verify-qr` → ترجّع `user_id` + بيانات العميل عند هذا التاجر.
- **تحمّل قطع النت (مهم — النت في المحلات بيفصل كتير):** قرّر سلوك الـ Scanner أوفلاين. **التوصية:** الـ `verify-qr` لازم تتم أونلاين (أمان)، فلو مفيش نت اعرض "لا يوجد اتصال — حاول مرة أخرى" بوضوح. (لا تخزّن نقاط محليًا وتزامن — بيفتح باب تلاعب.)

### 2.9 — ملف العميل (Customer Profile — بعد المسح) ⭐ قلب شغل الكاشير
- الأحرف الأولى/الصورة + اسم العميل + رقم العضوية.
- **بطاقة الحالة:** النقاط المتاحة + المستوى الحالي + "زيارة اليوم: لم تُسجّل / تم تسجيلها".
- عميل جديد على المتجر → شارة "عميل جديد 🎉" (الربط حصل تلقائيًا أول مسح).
- **حالة "مكافأة جاهزة":** لو وصل لهدف حملة → بانر "هذا العميل مؤهّل لمكافأة: [الاسم]" + زرار "تسليم المكافأة".
- **الأزرار الأساسية (4) — كبيرة وبأقل ضغطات:**

#### فلو "تسجيل زيارة"
- لسه مسجّلش النهاردة → تأكيد "تسجيل زيارة لـ [العميل]؟" → "تأكيد" → نجاح.
- سجّل قبل كده النهاردة → "تم تسجيل زيارة لهذا العميل اليوم بالفعل."
- **تقني:** `record-visit` تفرض قيد زيارة/يوم على الـ DB + تسجّل الفرع والموظف.

#### فلو "إضافة نقاط"
- أزرار سريعة: +10 / +20 / +50 / +100 · أو إدخال يدوي · حقل "سبب (اختياري)" · زرار "إضافة".
- بعد الإضافة: "تمت إضافة 50 نقطة" + رصيد العميل يتحدّث.
- **حدود:** أكبر من السقف → يترفض برسالة. (السقف يومي لكل موظف ولكل تاجر — يُفرض في `add-points`.)
- **تقني:** `earn` يزوّد available + lifetime، ويحدّث المستوى لو عدّى عتبة.

#### فلو "استبدال مكافأة"
- مكافآت المتجر: الاسم + السعر + هل العميل يقدر (نقاطه تكفي؟).
- اختيار → تأكيد "استبدال [المكافأة] مقابل 100 نقطة من رصيد العميل؟" → "تأكيد" → خصم من available فقط + نجاح.
- **تأكيد الطرفين (للمكافآت الكبيرة):** يطلب تأكيد من جهاز العميل (راجع شاشة "أرِ للكاشير" في تطبيق العميل + `confirm-redemption`).

#### فلو "تطبيق كوبون"
- إدخال كود أو اختيار من كوبونات العميل → التحقق من الصلاحية (`apply-coupon`) → تطبيق → نجاح.

### 2.10 — الإدارة (Management Hub)
تاب فيه دخول لكل أقسام الإعداد:

#### 2.10.أ — حملات الزيارة (Visit Campaigns)
- قائمة: الاسم + عدد الزيارات + المكافأة + سويتش تفعيل.
- إنشاء/تعديل: اسم الحملة · عدد الزيارات المطلوبة · اسم المكافأة · صورة (رفع) · وصف · سويتش "مفعّلة" · "حفظ".
- فاضي: "أنشئ أول حملة زيارات لتشجيع عملائك على العودة."

#### 2.10.ب — المكافآت (Rewards)
- قائمة: صورة + اسم + سعر بالنقاط + الكمية + مفعّلة.
- إنشاء/تعديل: اسم الجائزة · الصورة · الوصف · عدد النقاط · الكمية (أو "غير محدودة") · سويتش · "حفظ".

#### 2.10.ج — المستويات (Levels)
- قائمة: الاسم + العتبة (نقاط) + المكافأة، تصاعديًا.
- إنشاء/تعديل: اسم المستوى (برونزي/فضي...) · العتبة (lifetime) · وصف المكافأة.
- **تحذير في الشاشة:** "المستويات تعتمد على إجمالي النقاط (Lifetime) ولا تُخصم عند الوصول."

#### 2.10.د — الكوبونات (Coupons)
- قائمة: الكود + النوع + القيمة + الصلاحية + الاستخدامات.
- إنشاء/تعديل: الكود · النوع (نسبة/مبلغ/منتج مجاني) · القيمة · تاريخ البداية والنهاية · حد الاستخدام · حد لكل عميل.
- **ملاحظة في الشاشة:** ذكّر التاجر بالفرق بين الكوبون والمكافأة عشان ما يعملش تكرار يلخبط العميل.

#### 2.10.هـ — الفروع (Branches)
- قائمة: اسم الفرع + العنوان + مفعّل.
- إنشاء/تعديل: اسم الفرع · العنوان + تحديد الموقع على الخريطة (lat/lng) · نطاق إشعار القرب بالمتر (افتراضي 150).
- **ملاحظة:** ده اللي بيغذّي إشعار القرب وتقارير الفروع.

#### 2.10.و — الموظفين (Staff)
- قائمة: الاسم + الدور + الفرع + مفعّل.
- إضافة/تعديل: الاسم + رقم الجوال · الدور (مدير/كاشير/مدير فرع) · الفرع المرتبط · "حفظ".
- الصلاحيات حسب الدور (راجع الجدول فوق).

### 2.11 — التحليلات (Analytics)
- **فلتر الفترة** (اليوم/الأسبوع/الشهر) + **فلتر الفرع**.
- عملاء جدد + إجمالي العملاء.
- معدّل العودة (نسبة اللي زاروا مرتين+).
- رسم بياني: الزيارات على مدار الوقت (`fl_chart`).
- النقاط الموزّعة مقابل المُستبدلة.
- أكتر المكافآت استبدالًا (top 5).
- **نصيحة خبير:** خلّي المقاييس **محدّدة وثابتة** للنسخة الأولى (مش لوحة مفتوحة) — التجميع عبر الوقت أرخص وأسهل صيانة. استخدم Postgres **Materialized Views** + pg_cron للتحديث بدل حساب كل مرة.

### 2.12 — الإعلانات (Announcements) — بسيطة
- عنوان الإشعار + النص + زرار "إرسال".
- يستخدم بنية الـ Push نفسها (`send-push` لكل عملاء التاجر اللي عاملين opt-in).
- **ممكن تأجيله** لو عايز تخفّف النسخة الأولى.

### 2.13 — حسابي (Business Profile & Settings)
- **النشاط:** اللوجو + اسم النشاط + "تعديل بيانات المتجر" (الاسم/النوع/الوصف/اللوجو/العنوان/التواصل).
- **الاشتراك:** الخطة الحالية + تاريخ التجديد/انتهاء التجربة + "إدارة الاشتراك".
- **الإعدادات:** اللغة + الإشعارات.
- **الحساب:** تسجيل الخروج.

### 2.10.ز — الأسئلة (Questions) — جديد
- التاجر يضيف أسئلة يجاوبها العميل وياخد نقاط يحدّدها التاجر.
- قائمة الأسئلة: نص السؤال + النوع + النقاط + مفعّل/لأ + عدد الإجابات.
- إنشاء/تعديل سؤال:
  - النوع: **اختيار واحد** / **اختيار متعدد** / **نص حر**.
  - نص السؤال + وصف اختياري.
  - الخيارات (للاختيارات) — إضافة/حذف.
  - **النقاط الممنوحة** (التاجر يحدّدها) + سويتش "إجباري" + سويتش "مفعّل".
- **عرض الإجابات (في اللوحة):** لكل سؤال — توزيع الإجابات (للاختيارات: نِسَب/رسم)، وقائمة الإجابات النصية، وفلتر بالفرع/الفترة. (مصدرها `question_responses` المحميّة بـ RLS.)

### 2.14 — الإعدادات المتقدمة (Merchant Settings) ⭐ جديد — "أوبشن في كل حاجة"
شاشة فيها كل خيارات التاجر (تكتب في `merchant_settings`):
- **نطاق النقاط (Points Scope):** اختيار بين:
  - "مشترك بين كل الفروع" (`merchant`) — رصيد ومستوى واحد للعميل عند المتجر.
  - "منفصل لكل فرع" (`branch`) — رصيد ومستوى مستقل عند كل فرع.
  - تحذير في الشاشة: تغيير النطاق يؤثّر على طريقة احتساب المحافظ الجديدة.
- **تفعيل/تعطيل الميزات:** زيارات · نقاط · مكافآت · مستويات · كوبونات · إحالة · ميلاد · إشعار القرب · GPS · إعلانات (سويتشات).
- **حدود الأمان:** سقف النقاط للعملية · السقف اليومي للموظف · زيارة واحدة/يوم · تأكيد العميل على الاستبدال (+ عتبة) · مدة تجدّد الـ QR · عمر كود الاستلام.
- **الاكتساب:** نقاط لكل وحدة عملة (`earn_rate_per_currency`).
- **العلامة (White-Label):** لون أساسي مخصّص + اسم العلامة.

### 2.15 — لوحة صدارة المتجر (Store Leaderboard) — جديد
- ترتيب عملاء المتجر بإجمالي نقاطهم.
- **مبدّل العرض:** "الفرع الحالي" / "كل الفروع" — يستدعي `store_leaderboard(merchant, branch|null)`.
- لو نطاق النقاط "مشترك" → العرض بالفرع يرجع فاضي، فالأنسب "كل الفروع".
- تحترم خصوصية العميل (`leaderboard_opt_in`).

## حالات لازم تتعمل في كل شاشة (تذكير)
Loading (skeleton) · Empty (كل قائمة: حملات/مكافآت/مستويات/كوبونات/فروع/موظفين بنص ودود وزرار) · Error (رسالة + إعادة المحاولة) · Validation (تحت كل حقل، لحظي) · Success (toast/علامة صح) · Offline (bar علوي — والـ Scanner يوضّح إنه محتاج نت) · Permissions (كاميرا: شاشة تمهيد قبل الطلب) · انتهاء الجلسة.

## أداء (نصائح خبير لتطبيق التاجر)
- **سرعة المسح أولوية قصوى:** قلّل زمن `verify-qr` (استعلام واحد مفهرس). اعرض ملف العميل فورًا بعد التحقق.
- اضغط كل صورة قبل الرفع لـ Storage (`flutter_image_compress`) — توفير حجم وbandwidth.
- استخدم **Materialized Views** للتحليلات بدل حسابها لحظيًا.
- Firebase Analytics: تتبّع `scan_success`, `points_added`, `reward_delivered`, `visit_recorded` لقياس استخدام الكاشير.
- خلّي الأزرار الأربعة في ملف العميل **كبيرة** (الكاشير ممكن يكون مستعجل/بإيد واحدة).
